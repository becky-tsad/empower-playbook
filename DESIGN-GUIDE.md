# Empower Playbook — Page Blueprint

> Your guide to making every chapter feel like it belongs in the same book.
> Follow this when creating or editing any chapter.

---

## How Every Chapter is Built

Every chapter follows the same skeleton, in this exact order:

```
1. CHAPTER HEADER (colored banner)
2. OPENING HOOK (lead paragraph + framing box)
3. BODY SECTIONS (the meat — numbered H2s)
4. KEY TAKEAWAY (dark box at the bottom)
5. CHAPTER NAVIGATION (previous / next buttons)
```

That's it. No chapter should deviate from this order. Details for each below.

---

## 1. Chapter Header

The colored banner at the top of every chapter. It tells the reader where they are.

**Always includes:**
- Part label (e.g., "Part 2 — The Method")
- Chapter number badge
- Chapter title (large, white text)
- One-sentence subtitle describing what they'll learn
- Audience badges (E = Executives, F = Funders, I = Implementers)
- Read time badge

**Color is based on Part:**
| Part | Color | Hex | Chapters |
|------|-------|-----|----------|
| Part 1: The Case | Gray | #545454 | Ch 1–3 |
| Part 2: The Method | Blue | #46729d | Ch 4–6 |
| Part 3: Implementation | Green | #99B83C | Ch 7–8 |
| Part 3: Warning | Red | #dc2626 | Ch 9 only |
| Part 4: Scale | Navy | #2a4571 | Ch 10–12 |
| Part 5: Research | Gray | #545454 | Ch 15–17 |

---

## 2. Opening Hook

The first thing readers see after the header. Its job is to frame the chapter — tell them *why this matters* before diving into content.

**The pattern (use every time):**

1. **Lead paragraph** — 2–3 sentences in slightly larger text. Set the scene. What problem does this chapter address? Why should they care?

2. **Framing box** — A styled callout immediately after the lead paragraph that introduces the chapter's core concept, key question, or main framework. This is the "thesis statement" of the chapter.

**Example rhythm:**
> *Lead paragraph:* "Most rural programs fail not because of bad intentions, but because they copy urban models that don't fit. This chapter explores why."
>
> *Framing box:* A callout titled "The Core Problem" that defines the specific gap in 3–4 sentences.

---

## 3. Body Sections

The main content. This is where all the teaching, evidence, and guidance lives.

### Numbering

Every H2 section gets a number: `1.1`, `1.2`, `3.1`, `9.5`, etc. The first number is the chapter, the second is the section order. Use this format consistently — no exceptions.

### How Many Sections?

- **Foundational chapters** (Parts 1–2): 3–5 sections. These are building the case, so keep them focused.
- **Implementation chapters** (Parts 3–4): 5–8 sections. More granular because readers need actionable detail.
- **Research chapters** (Part 5): 3–5 sections. Dense but tightly organized.
- **Exception:** Chapter 9 (What Kills Programs) can have more sections since each failure pattern gets its own section.

### The Rhythm Rule

**Never go more than 3 paragraphs without a visual break.**

A "visual break" is any of these:
- A callout box
- A data box with statistics
- A blockquote / testimonial
- A bulleted or numbered list
- A table or comparison grid

This prevents walls of text and keeps readers scanning.

### Section Structure

Each H2 section should follow this general shape:

```
H2 Section Title (numbered)
├── Opening paragraph (what this section covers)
├── Content + at least 1 visual element
├── H3 subsections if needed (keep to 2–3 max)
└── Brief transition or connecting thought to next section
```

### Subsections (H3s)

- Use H3s to break up longer sections
- Maximum 3 H3s per H2 section — if you need more, the H2 is too broad
- H3s don't get numbers, just descriptive titles

---

## 4. Key Takeaway

Every chapter ends with a Key Takeaway box. No exceptions.

**What goes in it:**
- A "Key Takeaway" badge/label
- 1 paragraph summarizing the chapter's main insight (2–4 sentences)
- 1 highlighted sentence — the single most important thing to remember

**Think of it as:** If someone only reads the Key Takeaway, they should still understand the chapter's core message.

---

## 5. Chapter Navigation

Below the Key Takeaway, every chapter has navigation buttons:
- **Previous chapter** link (left side)
- **Next chapter** link (right side)
- Chapter 1 only shows "Next" — Chapter 17 only shows "Previous"

## Content Blocks: When to Use What

You have several styled content blocks. Here's when to reach for each one.

### Callout Box (colored border + background)

**Use when:** You want to highlight an approach, concept, or important note that supports the main text but deserves visual emphasis.

**Variants:**
| Type | When to use |
|------|-------------|
| **Green callout** | Empower methodology, recommended approaches, "do this" guidance |
| **Blue callout** | Research findings, evidence, citations, academic context |
| **Warning callout** | Pitfalls, common mistakes, things to avoid |
| **Dark callout** | Core concepts, foundational principles, "the big idea" |

**How long:** 1 short paragraph (2–4 sentences) or a short list. If it's longer than 5 sentences, it should probably be a regular section instead.

---

### Data Box (dark background with statistics)

**Use when:** You have hard numbers that prove a point — percentages, dollar amounts, ratios, outcomes.

**Format:** 2–4 metrics in a grid layout. Each metric has:
- The number (large, green text)
- A label (what the number measures)
- Optional: a comparison or context line

**Rule of thumb:** If you're writing "52% of participants..." in a paragraph, consider pulling that stat into a data box instead.

---

### Blockquote (green left border)

**Use when:** You're quoting a specific person — a practitioner, researcher, or community leader. Always include attribution (name + title/organization).

**Not for:** General statements or paraphrasing. Blockquotes should be actual quotes from real people.

---

### Testimonial Card (dark background with avatar)

**Use when:** A longer, more prominent quote that deserves its own visual space. Good for practitioner stories or leadership perspectives.

**Difference from blockquote:** Testimonial cards are bigger and more prominent. Use blockquotes for brief supporting quotes; use testimonial cards for quotes that are a centerpiece of the section.

---

### Lists (bulleted or numbered)

**Use when:** You're presenting 3+ parallel items — steps, options, criteria, examples.

**Bulleted lists** for items with no particular order.
**Numbered lists** for sequential steps or ranked items.

**Keep items short.** If each list item needs a full paragraph of explanation, consider using a different format (cards, subsections, or a table).

---

### Tables

**Use when:** You're comparing things across multiple dimensions — outcomes vs. benchmarks, traditional vs. Empower approach, before vs. after.

**Keep tables simple.** 2–4 columns max. If it's getting complex, break it into multiple smaller tables or use cards instead.

## Content Pacing by Chapter Type

Different chapters serve different purposes, so they have different visual "weights":

### Foundational Chapters (Parts 1–2: Chapters 1–6)
- **Tone:** Narrative, persuasive, building the case
- **Pacing:** Slower. More paragraphs, fewer cards/grids
- **Visual elements:** Mostly callouts, blockquotes, and data boxes
- **Goal:** Reader should feel convinced and informed

### Implementation Chapters (Parts 3–4: Chapters 7–12)
- **Tone:** Practical, actionable, specific
- **Pacing:** Faster. More lists, cards, grids, decision frameworks
- **Visual elements:** Cards, comparison tables, checklists, step-by-step lists
- **Goal:** Reader should feel equipped to act

### Research Chapters (Part 5: Chapters 15–17)
- **Tone:** Evidence-based, rigorous, analytical
- **Pacing:** Medium. Data-heavy with supporting narrative
- **Visual elements:** Data boxes, research callouts (blue), tables
- **Goal:** Reader should feel confident in the evidence

## Consistency Checklist

Run through this for every chapter before considering it done:

**Structure**
- [ ] Chapter header has: part label, chapter number, title, subtitle, audience badges, read time
- [ ] Opens with a lead paragraph followed by a framing box
- [ ] All H2 sections are numbered (X.1, X.2, etc.)
- [ ] Ends with a Key Takeaway box
- [ ] Has previous/next chapter navigation

**Content Rhythm**
- [ ] No more than 3 consecutive paragraphs without a visual break
- [ ] Each H2 section has at least 1 visual element (callout, data box, quote, list, or table)
- [ ] No H2 section has more than 3 H3 subsections
- [ ] Lists keep items short (not paragraph-length)

**Flow**
- [ ] Lead paragraph sets up "why this matters"
- [ ] Framing box introduces the core concept
- [ ] Sections flow logically — each builds on or follows from the previous
- [ ] Key Takeaway summarizes without introducing new information
- [ ] The highlighted sentence in the Key Takeaway could stand alone

**Technical**
- [ ] Page links to `../styles/design-system.css` (no inline style blocks for shared styles)
- [ ] Sidebar navigation marks this chapter as active
- [ ] All H2 sections have an `id` attribute for the table of contents
- [ ] Audience badges match the chapter's actual audience
